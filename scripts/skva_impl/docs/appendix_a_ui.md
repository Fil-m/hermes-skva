# Додаток A: Повний перелік UI елементів

Цей додаток містить повний перелік усіх елементів користувацького інтерфейсу (UI) у системі **SKVA** — включаючи CLI-команди, вихідні дані звітів, службові файли (gate, checkpoint), файли скілів, структуру `.skva/` та інші взаємодії з користувачем.

| # | Елемент | Тип | Опис | Розташування |
|---|--------|-----|------|-------------|
| 1 | `skva init` | CLI-команда | Ініціалізує новий SKVA-проект, створює `.skva/` та базові файли | Коренева папка проекту |
| 2 | `skva solo` | CLI-команда | Запускає режим Solo: один агент виконує весь цикл виробництва | Коренева папка проекту |
| 3 | `skva rada` | CLI-команда | Запускає режим Rada: колектив агентів приймає рішення через голосування | Коренева папка проекту |
| 4 | `skva agile` | CLI-команда | Запускає режим Agile Team: ітераційна розробка зі спринтами | Коренева папка проекту |
| 5 | `skva pipeline` | CLI-команда | Запускає режим Pipeline: лінійний потік етапів (аналог CI/CD) | Коренева папка проекту |
| 6 | `skva factory` | CLI-команда | Запускає режим Fabryka: масове виробництво продуктів за шаблонами | Коренева папка проекту |
| 7 | `skva discover` | CLI-команда | Аналізує існуючий код і генерує spec, архітектуру, документацію | Коренева папка проекту |
| 8 | `skva import` | CLI-команда | Імпортує зовнішній проект (наприклад, з GitHub) для подальшої обробки | Коренева папка проекту |
| 9 | `skva skills` | CLI-команда | Виводить список доступних скілів або інформацію про конкретний скіл | Будь-яка папка (глобальна) |
| 10 | `skva status` | CLI-команда | Показує поточний стан проекту, прогрес, активні агенти | Коренева папка проекту |
| 11 | `skva budget` | CLI-команда | Налаштовує або переглядає бюджет проекту (витрати на LLM, час, кроки) | Коренева папка проекту |
| 12 | `skva config` | CLI-команда | Налаштовує глобальні або локальні параметри (SSH, моделі, gateways) | Коренева папка проекту або `~/.skva/` |
| 13 | `skva logs` | CLI-команда | Перегляд логів агентів, включаючи помилки та внутрішні повідомлення | Коренева папка проекту |
| 14 | `skva reset` | CLI-команда | Скидає стан проекту (зберігає або видаляє артефакти) | Коренева папка проекту |
| 15 | `skva export` | CLI-команда | Експортує фінальний продукт у вигляді архіву, Dockerfile, або deploy-пакету | Коренева папка проекту |

| 16 | RunReport (вихід) | Звіт | Виводиться після кожного кроку: опис дії, витрати, статус, наступний крок | Термінал / лог-файл |
| 17 | Dashboard (вихід) | Звіт | Сумарний прогрес: % виконання, кількість кроків, витрати, агенти | Термінал / Telegram/Discord |

| 19 | `gate.run` | Файл (флаг) | Сигнал "почати виконання" — запускає наступний етап | `.skva/gate/` |
| 20 | `gate.pause` | Файл (флаг) | Сигнал "пауза" — тимчасово зупиняє виконання | `.skva/gate/` |

| 30 | `budget.log` | Файл (JSONL) | Послідовні записи витрат (токени, час, модель) | `.skva/logs/` |

| 32 | `plan.md` | Файл (Markdown) | План виконання проекту (етапи, відповідальні, терміни) | `.skva/artifacts/` |
| 33 | `feedback.user.md` | Файл (Markdown) | Відгук користувача, введений через інтерфейс | `.skva/input/` |
